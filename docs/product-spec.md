# CareerMap MVP 产品说明

## MVP 闭环

CareerMap v0.5 跑通以下链路：

1. 用户输入当前岗位、工作年限、城市、期望方向和可选目标岗位。
2. 用户可补充多段教育经历和工作经历摘要；这些信息进入能力判断。
3. 路径规划不强制依赖职业测评；目标岗位为空时，系统先根据当前岗位和方向给路径建议。
4. 系统生成纵向晋升路线与横向迁移岗位。
5. 用户选择或采用推荐目标后，系统输出已具备能力、待补能力、补充方式和转型周期。
6. 用户点击生成后，结果按路径地图、差距分析、在招职位的步骤依次展示，避免所有模块平铺。
7. 系统展示与目标岗位匹配的职位，优先使用授权 API/代理接口返回的真实职位；未配置时展示 AI 生成职位和平台搜索入口。
8. 用户可在页面底部选择职业测评，完成 8 题后获得职业倾向类型码、类型解释和岗位推荐。

## 数据边界

当前版本不直接抓取招聘网站页面。`src/planner.js` 中的 AI 生成职位结构保留了后续 API 替换所需字段：

- company
- title
- salaryMin / salaryMax / salaryRange
- city
- deadline
- applyUrl
- companySize
- tags

## 后续接口替换点

`server/index.js` 的 `/api/career-plan` 当前调用本地 `buildCareerMap()`。后续接入 GPT API 或 Claude API 时，可以保持响应结构不变，只替换生成器来源。

职位数据接入由 `server/services/jobs.js` 负责。当前支持配置以下授权接口：

- `BOSS_JOBS_ENDPOINT`
- `LIEPIN_JOBS_ENDPOINT`
- `JOB51_JOBS_ENDPOINT`
- `LINKEDIN_JOBS_ENDPOINT`

系统会传入 `keyword`、`city`、`platform` 查询参数，并把返回的职位标准化为前端统一字段。未配置或接口失败时，会保留 AI 生成职位兜底，并展示平台搜索入口。

## v0.3 改进点

- 页面改为左侧功能导航 + 工作区结构，路径规划和职业测评分离。
- 目标岗位保持选填，测评不再强制参与路径规划。
- 学历改为多段教育经历，支持学历/项目、学校/机构、专业/方向、时间。
- 职业测评输出类似 MBTI 的职业倾向类型码、维度结果和岗位推荐。
- 办公室主任的纵向晋升支持总经办、总经理助理、董事会秘书、幕僚长、副总经理/总经理等分支。

## v0.4 改进点

- 在招职位模块增加数据源状态提示。
- 增加 Boss、猎聘、前程无忧/51job、LinkedIn 搜索入口。
- 增加授权 API/代理接口接入层，支持真实职位数据标准化。
- 保留 AI 生成职位兜底，避免未配置 API 时页面空白。

## v0.5 改进点

- 主流程改为用户填写档案后点击“生成”，再按路径地图、差距分析、在招职位分步骤查看。
- 取消顶部功能切换，把职业适配测评移动到页面底部作为可选功能。
- 去掉页面上的数据源状态标签，职位区未接入授权 API 时展示为 AI 生成职位。
- 首屏保留结果占位，生成后确保路径、差距、职位都有内容可看。
